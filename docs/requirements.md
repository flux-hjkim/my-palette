# MyPalette Requirements

## Overview

MyPalette는 사용자가 자신의 취향, 기억, 성향, 관심사, 이상향을 색상 카드로 기록하고 카테고리별로 정리할 수 있는 웹 애플리케이션이다.

---

## Navigation

- 사용자는 첫 화면에서 서비스 제목과 소개 문구를 확인할 수 있다.
- 사용자는 첫 화면에서 MyPalette 화면으로 이동할 수 있다.
- 사용자는 상단 Navigation을 통해 첫 화면과 MyPalette 화면으로 이동할 수 있다.
- 사용자는 MyPalette의 각 카테고리 화면으로 이동할 수 있다.
- 사용자는 특정 카테고리 화면에서 다른 카테고리 화면으로 이동할 수 있어야 한다.

---

## Category

- 사용자는 5개의 큰 카테고리를 확인할 수 있다.
  - About Me
  - Taste
  - Interest
  - My Sides
  - Ideal Me

- 사용자는 각 카테고리의 대표 색상을 확인할 수 있다.
- 사용자는 각 카테고리의 이름과 비율 정보를 확인할 수 있다.
- 사용자는 카테고리를 선택하여 해당 카테고리의 세부 컬러 목록을 확인할 수 있다.

---

## Color List

- 사용자는 선택한 카테고리의 이름, 대표 컬러명, 설명을 확인할 수 있다.
- 사용자는 선택한 카테고리에 포함된 세부 컬러 목록을 확인할 수 있다.
- 사용자는 세부 컬러 카드에서 색상, 컬러명, HEX 코드, 제목을 확인할 수 있다.
- 사용자는 세부 컬러 카드를 선택하여 컬러 상세 정보를 확인할 수 있다.
- 사용자는 카테고리 화면에서 새로운 세부 컬러를 추가할 수 있다.

---

## Color Detail

- 사용자는 세부 컬러의 상세 정보를 Modal로 확인할 수 있다.
- 사용자는 컬러 상세 Modal에서 이름, 설명, keyword, 색상 정보를 확인할 수 있다.
- 사용자는 컬러 상세 Modal에서 해당 컬러를 수정할 수 있다.
- 사용자는 컬러 상세 Modal에서 해당 컬러를 삭제할 수 있다.
- 사용자는 Modal을 닫고 이전 화면으로 돌아갈 수 있다.
- Modal을 다시 열었을 때 이전 메뉴 상태가 남아있지 않아야 한다.
- Modal을 확인하는 동안 배경 화면이 함께 스크롤되지 않아야 한다.

---

## Editor

- 사용자는 새로운 컬러를 추가할 수 있다.
- 사용자는 기존 컬러를 수정할 수 있다.
- 컬러 추가와 수정은 별도 Editor Page에서 진행한다.
- Add와 Edit은 동일한 입력 구조를 사용한다.
- 사용자는 Editor Page에서 입력 중인 컬러 정보를 확인할 수 있다.
- 사용자는 Editor Page에서 선택한 색상을 preview로 확인할 수 있다.
- 사용자는 입력을 저장할 수 있다.
- 사용자는 입력을 취소하고 이전 화면으로 돌아갈 수 있다.
- 사용자는 저장 후 추가하거나 수정한 컬러 정보를 확인할 수 있어야 한다.

---

## Color Form

- 사용자는 컬러 이름을 입력할 수 있다.
- 사용자는 컬러 설명을 입력할 수 있다.
- 사용자는 컬러 keyword를 입력할 수 있다.
- 사용자는 컬러 색상을 선택할 수 있다.
- 사용자는 선택한 색상의 컬러명과 HEX 코드를 확인할 수 있다.
- 사용자는 입력 중인 설명의 글자 수를 확인할 수 있다.
- 컬러 설명은 최대 200자까지 입력할 수 있다.
- 사용자는 제목이 비어 있는 상태로 저장할 수 없다.
- 사용자는 설명이 비어 있는 상태로 저장할 수 없다.

---

## Keyword

- 사용자는 keyword를 입력할 수 있다.
- 사용자는 Enter 키로 keyword를 추가할 수 있다.
- 사용자는 공백 keyword를 추가할 수 없다.
- 사용자는 동일한 keyword를 중복 추가할 수 없다.
- 사용자는 추가된 keyword를 tag 형태로 확인할 수 있다.
- 사용자는 추가된 keyword를 삭제할 수 있다.

---

## Color Picker

- 사용자는 preset color 중 하나를 선택할 수 있다.
- 사용자는 선택한 색상이 preview에 반영되는 것을 확인할 수 있다.
- 사용자는 일부 대표 컬러를 먼저 확인할 수 있다.
- 사용자는 more 버튼을 통해 더 많은 컬러를 확인할 수 있다.
- 사용자는 컬러 선택 영역 바깥을 클릭하여 컬러 선택 UI를 닫을 수 있다.
- 컬러 선택 UI가 열려도 입력 화면의 레이아웃이 과도하게 흔들리지 않아야 한다.

---

## UI

- MyPalette 메인 화면은 큰 컬러 카드를 중심으로 구성한다.
- 카테고리 카드는 대표 색상이 충분히 잘 보이도록 표시한다.
- 카테고리 카드의 텍스트 정보는 컬러 카드 아래에 표시한다.
- 카테고리 상세 화면은 카테고리명, 대표 컬러명, 설명, 세부 컬러 목록 순서로 구성한다.
- 세부 컬러 카드는 색상 영역과 텍스트 정보가 함께 보이도록 구성한다.
- Add Color 카드도 세부 컬러 목록 안에서 함께 인식될 수 있도록 구성한다.
- Editor Page는 입력 영역과 컬러 preview 영역을 구분하여 구성한다.
- Modal 안의 정보는 제목, 설명, keyword의 계층이 구분되도록 표시한다.

---

## Interaction

- 카테고리 클릭 시 해당 카테고리 상세 화면으로 이동한다.
- 세부 컬러 카드 클릭 시 상세 Modal을 연다.
- Add Color 클릭 시 Editor Page로 이동한다.
- Edit 클릭 시 기존 컬러 정보를 수정할 수 있는 Editor Page로 이동한다.
- Delete 클릭 시 해당 컬러를 삭제한다.
- Save 클릭 시 입력한 컬러 정보를 저장한다.
- Cancel 클릭 시 입력을 취소하고 이전 화면으로 돌아간다.
- 컬러 선택 시 preview 색상이 변경된다.
- keyword 입력 후 Enter 입력 시 keyword tag가 추가된다.
- keyword tag 클릭 시 해당 keyword가 삭제된다.

---

## Data

- 새로운 세부 컬러를 추가하면 createdAt이 함께 저장되어야 한다.
- 기존 세부 컬러를 수정해도 createdAt은 유지되어야 한다.
- createdAt은 이후 Palette Log 기능에서 날짜별 기록을 보여주기 위한 데이터로 사용될 수 있다.

---

## Future Requirements

### Pieces of Me

- 사용자는 자신이 추가한 모든 컬러를 카테고리와 관계없이 한 화면에서 확인할 수 있다.
- 사용자는 컬러칩을 스티커 보드 형태로 볼 수 있다.
- 사용자는 컬러칩 hover 시 제목, 카테고리, keyword 정보를 확인할 수 있다.
- 사용자는 컬러칩 클릭 시 상세 정보를 확인할 수 있다.

### Palette Log

- 사용자는 자신이 추가한 컬러 기록을 월별 달력 형태로 확인할 수 있다.
- 사용자는 createdAt 기준으로 날짜별 컬러 기록을 확인할 수 있다.
- 사용자는 특정 날짜의 컬러칩을 클릭하여 상세 정보를 확인할 수 있다.

### Sticker Frame UI

- 사용자는 Pieces of Me 페이지에서 컬러칩을 빈티지 라벨 또는 스티커 형태로 확인할 수 있다.
- 컬러칩마다 다른 frame type이 적용될 수 있다.
- hover 시 컬러칩이 확대되고 관련 정보가 표시될 수 있다.
