# MyPalette v1.2 Completion Checklist

## Goal

MyPalette v1.2는 사용자가 기록한 컬러를
생성 날짜를 기준으로 달력 형태에서 확인할 수 있도록 한다.

v1.2의 목표는 복잡한 일정 관리 기능을 구현하는 것이 아니라,
기존 SubColor 데이터를 날짜별로 분류하고
사용자가 자신의 컬러 기록 흐름을 시각적으로 확인할 수 있는
기본 Palette Log 사용자 흐름을 완성하는 것이다.

---

## Required for v1.2

### Feature

- [x] Palette Log 메뉴로 이동할 수 있다.
- [x] Palette Log 전용 페이지가 표시된다.
- [x] 한 달의 날짜가 7열 달력 형태로 표시된다.
- [x] 각 SubColor가 생성된 날짜에 컬러 원으로 표시된다.
- [x] 같은 날짜에 여러 SubColor가 있으면 함께 표시된다.
- [x] 컬러 원을 클릭하면 상세 Modal이 열린다.
- [x] Modal에서 선택한 SubColor의 상세 정보를 확인할 수 있다.
- [x] Modal에서 SubColor를 수정할 수 있다.
- [x] 수정 후 Palette Log 페이지로 돌아온다.
- [x] Modal에서 SubColor를 삭제할 수 있다.
- [x] 삭제 결과가 Palette Log에 즉시 반영된다.
- [ ] 기록이 존재하는 달만 확인할 수 있다.
- [ ] 이전·다음 기록 월로 이동할 수 있다.

### Data

- [x] 카테고리별 subColors를 하나의 배열로 변환한다.
- [x] 각 SubColor가 원래 category 정보를 유지한다.
- [x] SubColor를 createdAt 기준으로 그룹화한다.
- [x] 날짜별 SubColor 배열을 생성한다.
- [x] 기존 colorGroups 데이터를 직접 변경하지 않는다.
- [ ] createdAt에서 기록이 존재하는 월 목록을 생성한다.

- [ ] 기록 월 목록을 날짜순으로 정렬한다.

### Calendar

- [x] 현재 연도와 월을 기준으로 마지막 날짜를 계산한다.
- [x] 해당 월의 첫 번째 요일을 계산한다.
- [x] 첫 번째 날짜 앞에 필요한 공백 셀을 생성한다.
- [x] 날짜와 공백을 포함한 달력 배열을 생성한다.
- [x] 날짜를 `YYYY-MM-DD` 형식의 key로 변환한다.
- [x] 날짜 key를 이용해 해당 날짜의 SubColor를 조회한다.
- [ ] 기록 월 이동 시 달력 날짜가 다시 계산된다.

## Done Criteria

아래 조건을 만족하면 MyPalette v1.2를 완료한 것으로 본다.

- 사용자가 Palette Log 페이지에 접근할 수 있다.

- 기존 MyPalette와 Pieces of Me 기능이 깨지지 않는다.

- `v1.2.0` tag와 GitHub Release가 생성되어 있다.
